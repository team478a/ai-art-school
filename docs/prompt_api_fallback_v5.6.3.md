# v5.6.3 プロンプトAPI障害時の生成継続

## 修正内容

- OpenAIのプロンプト整形APIが429、401、403、通信障害になっても、画像生成処理を停止しません。
- 外部APIの応答が空、またはJSONとして解析できない場合も、ローカルプロンプトへ自動的に切り替えます。
- Stability AIやGrokなど、選択された画像生成エンジンへ処理を継続します。
- 日本語の人物表現も検出し、ローカル補完時に顔・手・人体品質の制約を追加します。
- フォールバックの発生理由はPHPのエラーログへ短く記録します。

## 運用上の注意

OpenAIの利用枠を回復すると、より詳細な英語プロンプト変換へ自動的に戻ります。利用枠がない間も画像生成自体は継続できます。

更新前に失敗した依頼は、依頼詳細の「再生成」を押してください。CRONが利用できない場合は、依頼一覧の「次の1件を手動処理」でも再処理できます。
