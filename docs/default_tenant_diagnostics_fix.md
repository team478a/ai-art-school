# v5.3.6 標準テナントの診断表示修正

## 修正内容

標準アカウント、つまり `tenant_key = default` のクライアントは、既存システムの受け皿として使います。
そのため、新規クライアントと同じ「クライアント別初期設定」の完了率で評価すると誤表示になります。

このバージョンでは以下を修正しました。

- 標準アカウントの初期設定チェックを「共通設定で管理」と表示
- 標準アカウントは `/admin/settings` でAPI設定、LINE設定、公開ページ設定を確認する導線に変更
- データ分離診断で `ok / unassigned / message` を正しく返すように修正
- `tenant_id` と索引があり、未割当が0件ならOK表示
- 存在しない任意テーブルは「未使用」として扱う
- クライアント管理画面の日本語表示を整理

## 確認ポイント

1. `/admin/tenants` を開く
2. 標準アカウントの初期設定チェックが「共通設定で管理」と表示される
3. データ分離の準備状況で、未割当0件のテーブルがOKになる
4. 新規クライアントは従来通りクライアント別設定の完了率で表示される
