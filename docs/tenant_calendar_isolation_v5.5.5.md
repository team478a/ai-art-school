# v5.5.5 テナント別カレンダー分離

## 修正内容

- 管理カレンダーの開催予定を、現在操作中のテナントだけに限定しました。
- カレンダー内の予約数、承認数、承認待ち、キャンセル数、参加数も同じテナントだけを集計します。
- 削除済みの開催予定はカレンダーに表示しません。
- 新規開催予定の保存時に `tenant_id` を同じ INSERT で保存します。
- 非標準テナントで `tenant_id` カラムが欠けている場合は、全件表示せず空結果にする安全策を追加しました。

## 確認方法

1. クライアント管理から標準テナント以外へ切り替えます。
2. カレンダーを開き、標準テナントの開催予定が表示されないことを確認します。
3. 新しい開催予定を登録し、選択中テナントのカレンダーだけに表示されることを確認します。
4. 標準テナントへ戻り、標準テナントの予定だけが表示されることを確認します。
