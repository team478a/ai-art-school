# v5.1.9 Tenant Usage Summary

## 目的

SaaS化後に複数クライアントを一元管理しやすくするため、クライアント管理画面に運用サマリーを追加しました。

## 追加内容

- クライアント別のユーザー数
- 開催予定数と累計開催数
- 承認済み予約数と累計予約数
- 画像生成の完了数と失敗数
- 決済ログ数
- LINE送信ログ数
- 最終活動日時
- 注意項目の表示

## 注意項目

以下の状態は、管理画面上に確認事項として表示されます。

- 停止中
- ドメイン未設定
- ユーザーなし
- 今後の開催なし
- 生成失敗あり

## 安全性

途中導入の環境でも画面が落ちないように、対象テーブルやカラムが存在するかを確認してから集計します。
存在しないテーブルは 0 件として扱います。
