# v5.5.3 画像生成LIFF・リッチメニュー修正

## 修正内容

- クライアント別設定に「画像生成用LIFF ID」を追加しました。
- 画像生成ボタンは通常URLではなく `https://liff.line.me/{LIFF ID}` を開きます。
- 画像生成LIFFのEndpoint URLにクライアントキーを含め、別クライアントや標準アカウントへ切り替わる問題を防ぎます。
- 画像生成依頼の送信先にもクライアントキーを引き継ぎます。
- 画像生成だけの共通リッチメニュー画像を、大きな再生アイコンと `START` 表示に変更しました。
- オンライン生成型クライアントの初期設定チェックでは、予約用LIFFではなく画像生成用LIFFを確認します。

## LINE Developers設定

1. 対象クライアントの「クライアント別設定」を開きます。
2. 「クライアント専用URL」にある「画像生成 LIFF URL」をコピーします。
3. LINE DevelopersのLINE LoginチャネルでLIFFアプリを追加します。
4. Endpoint URLへコピーした画像生成 LIFF URLを登録します。
5. Scopeは `profile` と `openid` を有効にします。
6. 発行されたLIFF IDを「画像生成用LIFF ID」へ保存します。
7. リッチメニュー設定で「画像生成だけの設定にする」を実行します。
8. 「共通メニューを作成してLINEに表示」を実行します。

LINEトークへ「生成」と入力する操作は使用しません。リッチメニューの画像生成ボタンからLIFF画面を開きます。
