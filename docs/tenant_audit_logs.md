# v5.2.3 クライアント別ログイン監査ログ

## 目的

SaaS運用で、管理者・スタッフのログイン履歴をクライアント単位で確認できるようにします。

## 追加内容

- `admin_login_logs` に `tenant_id` と `tenant_key` を追加
- 管理者・スタッフのログイン成功、失敗、ログアウト時に所属クライアントを記録
- `/admin/logs` にクライアント絞り込みを追加
- 既存ログデータは保持

## 注意

- オーナー、スーパーオーナーは全体管理者のため、ログイン時のクライアントは未指定になる場合があります。
- 管理者・スタッフに所属クライアントが設定されている場合、そのクライアント情報がログに記録されます。
- `storage/logs/` のファイルログは全体ログとして表示されます。
