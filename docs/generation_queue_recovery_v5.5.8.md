# 画像生成キュー復旧 v5.5.8

## 症状

- 画像依頼が `received` のまま進まない
- 画像数が `0/8` のままになる
- `job_queue` の行がない、または古い処理状態が残っている

## 原因

cronがpendingキューの件数を先に確認していたため、`image_requests` に依頼があっても、対応するキュー行がない場合はワーカーが起動されませんでした。

## 修正

- ワーカー起動時に `received` の孤立依頼を検索
- 対応するpending/processingキューがなければ自動再登録
- cronは事前のpending件数判定を行わず、最初にワーカーを起動
- ワーカーは依頼の `tenant_id` へ切り替えてからAI API、LINE、生成制限を読み込む
- 5分以上processingで停止したキューをpendingへ戻す

## 適用後

cron 1回につき最大5件を処理します。8件滞留している場合は、通常2回以内のcron実行で処理対象になります。
