# v5.1.7 tenant scope hardening

SaaS化に向けて、クライアント別データ分離の漏れを減らす補修を行いました。

## 主な変更

- 教室一覧の予約数カウントを現在のクライアントに限定
- 教室の編集、中止、削除、一括承認、選択承認、予約削除を現在のクライアントに限定
- リマインダー送信対象を現在のクライアントの開催日・参加者・ユーザーに限定
- LINE webhook のユーザー取得/作成を現在のクライアントに限定
- リッチメニューのユーザー抽出、参加履歴判定、同期記録を現在のクライアントに限定
- ガチャ関連テーブルをテナント診断・tenant_id 付与対象に追加

## 追加で分離対象にしたガチャテーブル

- `gacha_rarities`
- `gacha_prizes`
- `gacha_entitlements`
- `gacha_purchase_interests`

## 注意

このアップデートは分離の強化フェーズです。次フェーズでは、ガチャ設定・抽選処理そのもののSQLにもさらに細かいテナント条件を追加します。
