# v5.1.4 Tenant Read Scope

## 目的

SaaS化に向けて、管理画面で表示される主要データを現在のクライアントに限定します。

v5.1.3で追加した `tenant_id` を利用し、クライアント別にダッシュボード・依頼一覧・統計が混ざらないようにします。

## 追加内容

- `TenantScopeService` を追加
- `tenant_id` カラムが存在するテーブルだけ自動で絞り込み
- ダッシュボードの依頼数・生成数・最近の依頼をテナント対応
- 依頼一覧と依頼詳細をテナント対応
- 月次レポート、売上、予約、生成数の集計をテナント対応
- 教室スケジュール一覧をテナント対応
- 新規開催日の登録後に現在のテナントIDを付与

## 安全設計

`tenant_id` カラムがまだ存在しないテーブルでは、従来通りのSQLで動作します。

そのため、v5.1.3のデータ分離準備が未実行の環境でも、画面がすぐ壊れないようにしています。

## 次フェーズ

- LINE Webhookから作成されるユーザー、予約、画像依頼への `tenant_id` 付与
- Stripe Webhookのテナント判定
- リッチメニュー、ガチャ、通知、予約カレンダーの完全テナント分離
