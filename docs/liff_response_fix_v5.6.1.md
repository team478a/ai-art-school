# v5.6.1 LIFF生成依頼の応答修正

## 修正内容

- 生成依頼の登録結果をJSONで先に返し、その後に画像生成処理を継続します。
- JSONの長さと依頼IDをレスポンスヘッダーへ設定し、共有サーバーでも画面側が成功を確定できるようにします。
- 出力バッファに混入した不要な文字を破棄してからJSONを返します。
- LIFF側はレスポンスを安全に解析し、依頼IDが返っている場合は登録成功として扱います。
- プロンプト生成ではOpenAIを使用し、Claude APIは呼び出しません。
- A案の画像を先に保存するため、B案の生成中も `4/8` のように進捗を確認できます。
- 認証エラーや不正なリクエストは長時間再試行せず、理由を表示して失敗状態へ移します。
- 5分以上停止した `analyzing`、`generating`、`uploading`、`sending` を処理キューへ戻せます。
- 停止処理を再開するときは、未完成のプロンプト・画像レコードを整理してから再生成します。

## 対象となる現象

生成依頼は管理画面に登録されているにもかかわらず、LIFF画面に次のエラーが表示される現象を修正します。

`The string did not match the expected pattern.`

このエラーは依頼登録の失敗ではなく、登録後のレスポンス解析で発生していました。
