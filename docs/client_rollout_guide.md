# AIアート教室 横展開導入ガイド v4.8.7

## 目的

このガイドは、AIアート教室システムを新しいクライアントへ展開するときの標準手順です。
まずはクライアント別コピー展開を前提にし、LINE公式、Stripe、AI API、DB、画像データをクライアントごとに分離します。

## 基本方針

- LINE公式アカウント、LINE Developers、Stripe、AI APIキーはクライアント所有にします。
- アップデートZIPは共通本体として利用し、顧客データと設定は上書きしません。
- `config/db.php`、`config/installed.lock`、`storage/`、`uploads/` は案件ごとに分離します。
- 業務フローは標準テンプレートを使い、案件ごとの差分は管理画面の「横展開・初期設定」で管理します。

## 導入手順

1. サーバーとドメインを用意します。
2. 新しいDBを作成し、専用DBユーザーを発行します。
3. システム本体をサーバーへアップロードします。
4. `config/db.php` を新しいDB情報に合わせます。
5. オーナー管理者を作成します。
6. 管理画面の「公開ページ設定」で会社情報、規約、特商法、プライバシーポリシーを確認します。
7. クライアント別の場合は「クライアント管理」→対象クライアントの「設定」、または操作中の左メニュー「このクライアント設定」でLINE公式ID、LIFF ID、リッチメニューを設定します。標準アカウントだけは従来通り「LINE設定」「API設定」を使います。
8. 管理画面の「API設定」でStripe、AI API、通知、生成制限を設定します。
9. 管理画面の「横展開・初期設定」で設定漏れを確認します。
10. 予約、承認、当日参加、画像生成、決済、Webhookをテストします。

## LINE設定チェック

- LINE公式アカウントはクライアント名義で作成します。
- Messaging APIチャネルの Channel Secret と Channel Access Token を設定します。
- 予約用LIFFと購入用LIFFは分けて作成します。
- LIFF Endpoint URL は本番ドメインに合わせます。
- リッチメニューはユーザー状態別に切り替える前提で設定します。
  - 初回ユーザー
  - 参加済みユーザー
  - 回数券ユーザー
  - サブスクユーザー

## Stripe設定チェック

- Stripeアカウントはクライアント所有にします。
- テスト環境と本番環境のキーを混在させないでください。
- 月額サブスク、年額サブスク、回数券、一回払いは、それぞれStripeの商品価格で作成します。
- システム側には `price_` で始まる Price ID を設定します。`prod_` で始まる Product ID は設定しません。
- Webhook URL は `/stripe/webhook` を設定します。
- Webhookイベントは最低限以下を有効にします。
  - `checkout.session.completed`
  - `invoice.paid`
  - `customer.subscription.updated`
  - `customer.subscription.deleted`
  - `payment_intent.succeeded`
  - `payment_intent.payment_failed`

## AI API設定チェック

- 画像生成APIキーはクライアント所有または契約上合意した管理者所有にします。
- Stability AI、OpenAI、Grokなど複数エンジンを使う場合は、優先順位と残高不足時の切替条件を確認します。
- 顔・手の崩れを避けるため、標準プロンプト、禁止ワード、品質ガードを初期値として有効にします。
- 失敗時は生成数を消費しない設定を確認します。

## 標準フロー

標準フローは以下です。

1. 参加者がLINE登録します。
2. 予約カレンダーから教室を予約します。
3. 管理者が承認します。
4. 当日に会場案内またはURLを送ります。
5. 参加者が参加ボタンを押します。
6. 出席確認後に画像生成できます。
7. 有料条件の場合は回数券、サブスク、初回無料、現金支払いを判定します。

## 権限設計

- オーナー：API設定、LINE設定、Stripe設定、公開ページ設定、アップデート、ガチャ設定、横展開設定
- 管理者：教室運営、予約、出席、参加者、売上確認、LINE配信
- スタッフ：当日受付、QR、出席確認、予約確認

## 動作確認チェックリスト

- 新規ドメインで管理画面にログインできる
- 初期設定チェックに未設定項目が表示される
- 友だち追加QRからLINE登録できる
- 予約LIFFで予約できる
- 管理画面で承認できる
- 承認後の案内が正しく送信される
- 当日参加ボタンで出席確認できる
- 画像生成が成功する
- 画像がLINEで受信できるURL形式になっている
- Stripe決済が成功し、Webhookで反映される
- 回数券、月額、年額、一回払い、初回無料が正しく判定される
- 管理者とスタッフで表示メニューが分離されている
- アップデート後も `config/`、`storage/`、`uploads/` が消えない

## 将来のSaaS化に向けたメモ

現時点ではクライアント別コピー展開を優先します。
将来的にSaaS化する場合は、DBのテナント分離、ドメイン分離、LINE/Stripeの接続管理、請求管理、利用量集計を共通管理する設計へ移行します。
