# v5.1.1 クライアント別設定台帳

## 目的

SaaS化に向けて、クライアントごとの設定値を管理できる台帳を追加しました。

## 追加内容

- `/admin/tenants/{id}/settings` を追加
- クライアント別に以下の設定値を保存可能
  - 基本情報
  - 公開ページ・法務表記
  - LINE・LIFF
  - Stripe・料金
  - AI生成
  - 運用ルール
- `tenant_settings` テーブルへ保存
- クライアント一覧に「設定」ボタンを追加

## 重要

このバージョンでは、クライアント別設定の保存領域と管理画面を追加します。
既存の本番処理はまだ全体共通の `Settings::get()` を優先します。

次のフェーズで、アクセス中のドメインや選択中クライアントに応じて、
LINE、Stripe、AI生成、公開ページへクライアント別設定を反映します。

## 適用後の確認

1. `/admin/tenants` を開く
2. クライアント一覧の「設定」をクリックする
3. LINE、Stripe、AI、公開ページ、運用ルールの入力欄が表示される
4. 保存後、再表示して値が残っていることを確認する
