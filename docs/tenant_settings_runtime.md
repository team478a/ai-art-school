# v5.1.2 クライアント別設定の実行時反映

## 目的

SaaS化に向けて、保存済みのクライアント別設定を既存処理から参照できるようにしました。

## 変更内容

- `config/settings.php` をテナント対応版に更新
- 現在アクセス中のドメインから `tenants.primary_domain` を判定
- 一致する有効クライアントがある場合、その `tenant_settings` を `system_settings` より優先
- 一致するクライアントがない場合は既定クライアント、または従来の全体設定を使用
- `Settings::get()` を使っている既存のLINE、Stripe、AI、公開ページ処理が、段階的にクライアント別設定を読めるようになります

## 保存時の挙動

- 既定クライアントでは従来通り `system_settings` に保存します
- 既定ではないクライアントのドメインでアクセスしている場合、`Settings::set()` は `tenant_settings` に保存します

## 注意

この更新はSaaS化の第2段階です。
データ本体、予約、ユーザー、決済履歴、画像履歴の完全なテナント分離は次フェーズで行います。

## 適用後の確認

1. `/admin/tenants` でクライアントのメインドメインを設定
2. `/admin/tenants/{id}/settings` でLINE、Stripe、AIなどを設定
3. そのドメインでアクセスし、API設定やLINE設定が対象クライアントの値で動くことを確認
4. 既定クライアントでは従来設定が引き続き使えることを確認
