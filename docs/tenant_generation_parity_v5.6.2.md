# v5.6.2 テナント画像生成の整合性修正

## 原因

画像生成ワーカー自体はデフォルトと追加テナントで共通ですが、追加テナントはAPIキー、画像エンジン、品質、生成上限を独立管理しています。

従来のワーカーは `images_per_pattern` だけを使い、`max_images_per_request` を無視していました。そのため、追加テナントで最大4枚に設定していても、A/B各4枚の合計8枚を生成していました。OpenAI高品質で8枚を処理すると共有サーバー上で長時間処理となり、`generating` のまま停止しやすい状態でした。

## 修正内容

- テナントの `max_images_per_request` を実処理へ反映
- 最大4枚の場合はA/B各2枚として生成
- 1枚設定の場合はAパターンだけを生成
- 生成API呼び出しの前後でジョブの生存時刻を更新
- 401、403、400などの設定エラーは無駄に再試行せず失敗として確定
- エラーにテナント、エンジン、モデル、品質、生成上限、APIキー設定有無を付加
- 依頼一覧の固定 `/8` 表示をテナント設定値へ変更
- 依頼詳細に実際の生成設定を表示
- LINEの受付メッセージを実際の生成枚数へ合わせる

## 更新後の確認

1. 対象クライアントへ切り替える
2. 依頼詳細を開き「このクライアントで実際に使う生成設定」を確認する
3. `generating` で停止した依頼は、5分経過後に「今すぐ手動処理」を押す
4. 最大4枚設定の場合、一覧が `0/4` から進み、最終的に `4/4` になることを確認する
