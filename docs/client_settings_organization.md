# クライアント別設定の整理 v5.4.7

## 目的

SaaS運用では、標準アカウント用の `API設定` と、クライアントごとの設定を分けて管理します。

クライアントで操作中は `API設定` を表示せず、左メニューの `このクライアント設定` から以下を管理します。

- 基本情報
- 公開ページ・法務表記
- LINE公式ID、Messaging API、LIFF ID
- リッチメニュー出し分け
- Stripe・料金プラン
- AI生成API・エンジン
- オンライン生成、予約承認、生成数などの運用ルール

## 標準アカウントとの違い

標準アカウントは既存システムの受け皿です。
標準アカウントの共通設定は従来通り `API設定`、`LINE設定`、`公開ページ設定` で管理します。

新しく追加したクライアントは、クライアント一覧から `設定` を開き、クライアント別設定に入力します。

## 予約なしオンライン生成の設定

予約を使わずに、LINEからオンライン生成だけを使う場合は、クライアント別設定の `運用ルール` で以下を設定します。

1. `運用タイプ` を `オンライン生成型` にする
2. `オンライン生成を有効にする` をオンにする
3. `生成申請の受付方式` を `受付時間内なら生成可能` または `常時受付` にする
4. 生成可能開始日、終了日、曜日、時間帯、期間内の最大生成依頼数を設定する
5. リッチメニューに生成ボタンと購入ボタンを設定する

## リッチメニュー

クライアントごとのLINE公式アカウントで作成した Rich Menu ID を登録します。

- 初回・未参加ユーザー用
- 参加済みユーザー用
- 回数券ユーザー用
- サブスクユーザー用

保存後、既存ユーザーへ反映したい場合はクライアント別設定画面の `リッチメニュー同期` を実行します。
